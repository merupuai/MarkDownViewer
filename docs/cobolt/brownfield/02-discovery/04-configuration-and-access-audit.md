## Authentication & Authorization

**The application has no auth model.**

- No user accounts, no login, no session, no JWT, no OAuth, no API keys consumed at runtime
- No middleware guards (because no HTTP server)
- No RBAC / ABAC matrix (because no operations are gated)
- Database-level permissions: n/a (no database)

The only "gate" is the first-run EULA dialog (`ensureEulaAccepted`). Marker presence = consent. This is a license enforcement gate, not an authentication gate.

## Access Audit Summary

| Surface | Auth Required? | Authz Mechanism | Threat |
|---|---|---|---|
| File read (`readFile` RPC) | No | OS file ACLs (Bun process runs as user) | A hostile markdown file can request `resolveImage` for any file readable by the user → `data:image/...;base64,...` data URL exfiltrated into rendered HTML |
| Folder open | No | OS file ACLs | None notable |
| Folder search | No | Bounded by depth/size/file caps | DoS-grade only (regex compile from user input is `escape`d before RegExp construction) |
| External URL open (`openExternal`) | No | OS browser ACLs | Limited — `Bun.spawn(["open", url])` / `cmd /c start "" url` could shell-out if URL contains shell metacharacters; but URLs come from rendered markdown links sanitized by DOMPurify and the actual exec uses array argv (not shell string), so injection is contained |
| Reveal in Finder/Explorer | No | OS file ACLs | None notable |
| Print | No | OS print spooler | None notable |
| Export to HTML | No | User-chosen folder | Filename sanitized to `[A-Za-z0-9._-]` (`src/bun/index.ts:488`); contents are the rendered HTML which has been DOMPurify-sanitized at render time |
